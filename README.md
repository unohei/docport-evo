# DocPort

## 送らない共有。置くだけ連携。

DocPort は、医療機関における  
FAX・紙・電話に依存した情報共有を、

**書類をデータとして扱う形に再構成する共有基盤**です。

---

> 「送る」のではなく、共通の場所に「置く」  
> 書類はそのままではなく、データとして扱う

---

従来の業務では、

- 送信（FAX）
- 受信（紙）
- 確認（電話）
- 管理（人）

といった分断された流れが発生していました。

DocPortはこれらを一つにまとめ、  
書類を中心とした業務を「管理できる状態」に変えます。

---

## 主な機能

### 置く（Send）

- PDFをドラッグ＆ドロップで配置
- 宛先を選択して「置く」だけのシンプル操作
- カメラによる書類撮影（台形補正・PDF化）
- 送信ではなく「状態として残る」設計

---

### 受け取る（Inbox）

- 自院宛ての書類を一覧表示
- 未対応 / 対応中 / 完了 などの状態を可視化
- PDF・画像のアプリ内プレビュー対応
- 担当者ごとの管理が可能

---

### 記録（Records）

- 自院が配置した書類の履歴を管理
- 状態に応じた取消・管理
- 書類単位での追跡が可能

---

## 特徴

- 「送信」ではなく「配置」による共有
- 書類をデータとして扱う設計（OCR + AI解析）
- 状態と担当者を明確にすることで業務を可視化
- FAX送受信との併用が可能（既存運用との共存）
- 紙・PDF・FAXを一つの流れで扱える

---

## 技術構成

- Frontend：React / Vite / Cloudflare Pages  
- Backend：FastAPI / Render  
- Auth / DB：Supabase（Auth / PostgreSQL）  
- Storage：Cloudflare R2（Presigned URL）  
- 認証：JWT / Row Level Security（RLS）  
- 外部連携：FAX受信Webhook  
- AI：OCR + LLMによる構造化処理
